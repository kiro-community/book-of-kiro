# **Kiro Autonomous Agent 自主智慧代理**

Kiro Autonomous Agent 是 Kiro 平台的前沿元件，能夠獨立處理開發任務，保持上下文並從每次互動中學習。

{{% hint info %}}
Autonomous Agent 目前正在向 Kiro Pro、Pro+ 和 Power 使用者推出預覽版。團隊存取需要透過候補名單邀請。
{{% /hint %}}

## **🤖 核心特性**

### **自主工作能力**
讓 Kiro 自主處理開發任務，您可以保持專注或暫時離開，工作會繼續進行。

### **上下文保持**
回到任務時可以從中斷的地方繼續，無需重新建構上下文。

### **跨儲存庫執行**
建立任務後，Kiro 會處理跨所有儲存庫的規劃和實施。

### **並行執行不阻塞工作流程**
在隔離的沙箱環境中執行任務並開啟 Pull Request 供審查，工作在您專注於當前任務時繼續進行。

### **持續的上下文管理**
在任務、儲存庫和 Pull Request 之間保持上下文，並使用您的審查回饋來塑造未來的變更，使其更好地匹配您和團隊建構軟體的方式。

### **一次性多儲存庫變更**
規劃一次變更，讓 Kiro 建立協調的編輯和 Pull Request，將相關更新一起落地到各個儲存庫，而不是逐個處理。

## **👥 團隊協作功能**

Kiro Autonomous Agent 為團隊帶來上下文和協調，與您的工程團隊一起工作以放大影響力：

### **🚀 更快地協同發布**
在團隊中並行執行開發工作，使發布能夠以更少的停滯和協調拖累向前推進。

### **🔧 跨技術堆疊工作**
連接團隊的儲存庫、流水線和協作工具，在工作進展時將計畫、變更和理由保持在一個共享上下文中。

### **🎯 保護專注時間**
處理常規修復、後續工作和狀態更新，使工程師面臨更少的中斷，保持更長的深度工作時間區塊。

### **📚 從團隊中學習**
持續從您的程式碼庫、工單和回饋中學習，以更好地與團隊的開發模式保持一致。

## **🔐 安全與隔離**

- **沙箱環境**：所有任務在隔離的環境中執行，確保安全性
- **Pull Request 工作流程**：變更透過 PR 提交，保持程式碼審查流程
- **權限控制**：遵循現有的儲存庫權限和存取控制

## **📋 使用場景**

### **個人開發者**
- 自動化重複性開發任務
- 跨專案維護一致性
- 在您專注於核心功能時處理輔助任務

### **團隊協作**
- 協調多儲存庫功能開發
- 自動化程式碼維護和更新
- 減少團隊間的協調開銷

### **企業級應用**
- 大規模程式碼庫管理
- 合規性和安全性檢查
- 跨團隊標準化實施

## **🚀 開始使用**

{{% hint warning %}}
Autonomous Agent 目前處於預覽階段，僅對符合條件的使用者開放。
{{% /hint %}}

1. **檢查資格**：確認您擁有 Kiro Pro、Pro+ 或 Power 訂閱
2. **申請存取**：透過 Kiro IDE 或官網申請預覽存取
3. **設定任務**：在 Kiro IDE 中建立您的第一個自主任務
4. **監控進度**：透過 Kiro 介面追蹤任務執行狀態

## **💡 最佳實踐**

### **任務定義**
- 提供清晰、具體的任務描述
- 包含必要的上下文和約束條件
- 設定合理的完成時間預期

### **程式碼審查**
- 及時審查 Autonomous Agent 建立的 Pull Request
- 提供具體的回饋以改進未來的輸出
- 保持團隊程式碼標準的一致性

### **監控與調整**
- 定期檢查任務執行狀態
- 根據結果調整任務複雜度
- 利用學習回饋最佳化工作流程
