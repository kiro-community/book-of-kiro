# **Kiro Web 浏览器开发环境**

Kiro Web 是 Kiro 平台的浏览器端开发环境，提供两种工作模式：Vibe 模式（交互式协作开发）和 Autonomous 模式（自主任务处理）。无需本地环境配置，在浏览器中即可完成完整的开发工作流。

{{% hint info %}}
Kiro Web 目前正在向 Kiro Pro、Pro+ 和 Power 用户推出预览版。团队访问需要通过候补名单邀请。
{{% /hint %}}

## **🌐 核心特性**

### **双模式开发**

Kiro Web 提供两种工作模式，适应不同的开发场景：

- **Vibe 模式**：交互式协作开发，与 AI 实时对话，共同构建功能。适合探索性开发、原型设计和需要频繁交互的任务。
- **Autonomous 模式**：自主任务处理，分配任务后 Kiro 独立完成规划和实施。适合明确定义的任务、重复性工作和跨仓库变更。

### **浏览器端开发环境**

无需本地安装或配置，在浏览器中即可获得完整的开发体验：

- 完整的代码编辑和执行环境
- 终端访问和命令执行
- 文件系统浏览和管理
- 实时预览和调试

### **沙盒执行环境**

所有任务在安全隔离的沙盒环境中运行：

- **可配置网络访问**：根据需要设置网络权限级别
- **隔离执行**：每个任务在独立环境中运行，互不干扰
- **安全保障**：代码执行与生产环境完全隔离

### **GitHub/GitLab 仓库集成**

连接您的代码仓库，实现无缝工作流：

- 直接连接 GitHub 和 GitLab 仓库
- 支持跨仓库规划和实施
- 自动创建 Pull Request 供团队审查
- 遵循现有的仓库权限和访问控制

### **PR 工作流**

所有代码变更通过 Pull Request 提交：

- 自动创建结构化的 PR 描述
- 保持团队代码审查流程
- 支持审查反馈的持续迭代
- 协调多仓库的关联 PR

## **👥 团队协作功能**

Kiro Web 为团队带来上下文和协调，与您的工程团队一起工作以放大影响力：

### **🚀 更快地协同发布**
在团队中并行运行开发工作，使发布能够以更少的停滞和协调拖累向前推进。

### **🔧 跨技术栈工作**
连接团队的仓库、流水线和协作工具，在工作进展时将计划、更改和理由保持在一个共享上下文中。

### **🎯 保护专注时间**
处理常规修复、后续工作和状态更新，使工程师面临更少的中断，保持更长的深度工作时间块。

### **📚 从团队中学习**
持续从您的代码库、工单和反馈中学习，以更好地与团队的开发模式保持一致。

## **🔐 安全与隔离**

- **沙盒环境**：所有任务在隔离的环境中执行，确保安全性
- **可配置网络**：根据任务需求配置网络访问级别
- **Pull Request 工作流**：更改通过 PR 提交，保持代码审查流程
- **权限控制**：遵循现有的仓库权限和访问控制

## **📋 使用场景**

### **个人开发者**
- 使用 Vibe 模式进行探索性开发和原型构建
- 使用 Autonomous 模式自动化重复性开发任务
- 跨项目维护一致性
- 在您专注于核心功能时处理辅助任务

### **团队协作**
- 协调多仓库功能开发
- 自动化代码维护和更新
- 减少团队间的协调开销
- 并行处理多个开发任务

### **企业级应用**
- 大规模代码库管理
- 合规性和安全性检查
- 跨团队标准化实施

## **🚀 开始使用**

{{% hint warning %}}
Kiro Web 目前处于预览阶段，仅对符合条件的用户开放。
{{% /hint %}}

1. **检查资格**：确认您拥有 Kiro Pro、Pro+ 或 Power 订阅
2. **访问 Kiro Web**：通过浏览器访问 [kiro.dev](https://kiro.dev) 并登录
3. **连接仓库**：连接您的 GitHub 或 GitLab 仓库
4. **选择模式**：根据任务选择 Vibe 模式或 Autonomous 模式
5. **开始开发**：在浏览器中完成开发工作

## **💡 最佳实践**

### **模式选择**
- 探索性开发和原型设计使用 Vibe 模式
- 明确定义的任务和重复性工作使用 Autonomous 模式
- 复杂任务可以先用 Vibe 模式规划，再用 Autonomous 模式执行

### **任务定义**
- 提供清晰、具体的任务描述
- 包含必要的上下文和约束条件
- 设置合理的完成时间预期

### **代码审查**
- 及时审查 Kiro Web 创建的 Pull Request
- 提供具体的反馈以改进未来的输出
- 保持团队代码标准的一致性

### **网络配置**
- 根据任务需求配置合适的网络访问级别
- 敏感项目使用受限网络模式
- 需要外部依赖的任务启用适当的网络访问
