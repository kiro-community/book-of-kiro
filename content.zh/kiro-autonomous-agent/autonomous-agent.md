# **Kiro Autonomous Agent 自主智能体**

Kiro Autonomous Agent 是 Kiro 平台的前沿组件，能够独立处理开发任务，保持上下文并从每次交互中学习。

{{% hint info %}}
Autonomous Agent 目前正在向 Kiro Pro、Pro+ 和 Power 用户推出预览版。团队访问需要通过候补名单邀请。
{{% /hint %}}

## **🤖 核心特性**

### **自主工作能力**
让 Kiro 自主处理开发任务，您可以保持专注或暂时离开，工作会继续进行。

### **上下文保持**
回到任务时可以从中断的地方继续，无需重新构建上下文。

### **跨仓库执行**
创建任务后，Kiro 会处理跨所有仓库的规划和实施。

### **并行执行不阻塞工作流**
在隔离的沙盒环境中运行任务并打开 Pull Request 供审查，工作在您专注于当前任务时继续进行。

### **持续的上下文管理**
在任务、仓库和 Pull Request 之间保持上下文，并使用您的审查反馈来塑造未来的更改，使其更好地匹配您和团队构建软件的方式。

### **一次性多仓库更改**
规划一次更改，让 Kiro 创建协调的编辑和 Pull Request，将相关更新一起落地到各个仓库，而不是逐个处理。

## **👥 团队协作功能**

Kiro Autonomous Agent 为团队带来上下文和协调，与您的工程团队一起工作以放大影响力：

### **🚀 更快地协同发布**
在团队中并行运行开发工作，使发布能够以更少的停滞和协调拖累向前推进。

### **🔧 跨技术栈工作**
连接团队的仓库、流水线和协作工具，在工作进展时将计划、更改和理由保持在一个共享上下文中。

### **🎯 保护专注时间**
处理常规修复、后续工作和状态更新，使工程师面临更少的中断，保持更长的深度工作时间块。

### **📚 从团队中学习**
持续从您的代码库、工单和反馈中学习，以更好地与团队的开发模式保持一致。

## **🔐 安全与隔离**

- **沙盒环境**：所有任务在隔离的环境中执行，确保安全性
- **Pull Request 工作流**：更改通过 PR 提交，保持代码审查流程
- **权限控制**：遵循现有的仓库权限和访问控制

## **📋 使用场景**

### **个人开发者**
- 自动化重复性开发任务
- 跨项目维护一致性
- 在您专注于核心功能时处理辅助任务

### **团队协作**
- 协调多仓库功能开发
- 自动化代码维护和更新
- 减少团队间的协调开销

### **企业级应用**
- 大规模代码库管理
- 合规性和安全性检查
- 跨团队标准化实施

## **🚀 开始使用**

{{% hint warning %}}
Autonomous Agent 目前处于预览阶段，仅对符合条件的用户开放。
{{% /hint %}}

1. **检查资格**：确认您拥有 Kiro Pro、Pro+ 或 Power 订阅
2. **申请访问**：通过 Kiro IDE 或官网申请预览访问
3. **设置任务**：在 Kiro IDE 中创建您的第一个自主任务
4. **监控进度**：通过 Kiro 界面跟踪任务执行状态

## **💡 最佳实践**

### **任务定义**
- 提供清晰、具体的任务描述
- 包含必要的上下文和约束条件
- 设置合理的完成时间预期

### **代码审查**
- 及时审查 Autonomous Agent 创建的 Pull Request
- 提供具体的反馈以改进未来的输出
- 保持团队代码标准的一致性

### **监控与调整**
- 定期检查任务执行状态
- 根据结果调整任务复杂度
- 利用学习反馈优化工作流程
